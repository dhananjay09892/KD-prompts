# System Designs Index

Pick the pattern closest to your project. Adapt minimally — don't reinvent.

| Pattern | File | Use When |
|---------|------|----------|
| SaaS Platform | `saas-platform.md` | Multi-tenant web app, auth, billing, dashboards |
| Frontend Architecture | `frontend-architecture.md` | React/Next.js app requiring scale + DX |
| Scalable Backend | `backend-scalable.md` | REST/GraphQL API with growth in mind |
| AI Integration | `ai-integration.md` | Adding LLMs, embeddings, RAG to any project |
| Realtime App | `realtime.md` | Chat, live updates, collaborative features |
| CLI Tool | `cli-tool.md` | Node/Python developer CLI with plugins |

## How to Use

1. Open the relevant `.md` file
2. Read the **Constraints** section first — confirms it matches your case
3. Copy the **Stack Decision** table directly into your `ARCHITECTURE.md`
4. Use the **Folder Structure** as your project skeleton
5. Reference **Security Checklist** before going live
